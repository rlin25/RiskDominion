## 1. MAP AND TERRITORY RENDERING

### 1.1 Map Container

- Full viewport: `width: 100vw`, `height: 100vh`
- Background: `#1a1d1c` (bg-ocean)
- D3.js rendering using Mercator or Equal Earth projection
- SVG element fills the container
- Pan: D3 `zoom()` behavior, translate on drag
- Zoom: D3 `zoom()` behavior, scale extent `[1, 4]`, scroll wheel
- Edge fades: four `linearGradient` overlays at viewport edges, `#1a1d1c` at 0% to transparent at 8% inset

### 1.2 GeoJSON Data

- 12 territories defined as GeoJSON `FeatureCollection`
- Each feature has `properties: { territory_id, name, continent }`
- Continents: Americas (territories 1-4), Europe-Africa (5-8), Asia-Oceania (9-12)
- Territory shapes are irregular polygons approximating real geographic borders
- Rendered via D3 `geoPath()` with the configured projection

### 1.3 Territory Base Rendering

- Fill: `#2d302e` (bg-landmass)
- Stroke: `#3a3f3c` (border-subtle), `stroke-width: 1px`
- Hover: stroke transitions to `#c5c9c6` (text-primary), `stroke-width: 2px`, transition `200ms ease-out`
- Cursor: `pointer` on hover

### 1.4 Territory Patterns (Procedural D3)

Patterns are generated by `patternRenderer.ts`. Each dimension owner contributes one pattern layer. Patterns overlay additively.

**Military Pattern:**
- Type: contour-parallel lines
- Generated by creating inset paths from the territory perimeter inward at 6px intervals
- Stroke: owner's player color at 40% opacity, `stroke-width: 1px`

**Economic Pattern:**
- Type: jittered dot grid
- Generate a grid of points within the territory bounding box at 8px spacing
- Apply random jitter of ±2px to each point
- Clip to territory path
- Fill: owner's player color at 40% opacity, circle `r: 0.75px`

**Cultural Pattern:**
- Type: crosshatch
- Generate lines at 45° and -45° from the territory centroid
- Spacing: 10px between parallel lines
- Clip to territory path
- Stroke: owner's player color at 40% opacity, `stroke-width: 1px`

**Covert Pattern:**
- Type: Poisson disk distribution of circles
- Generate points using Poisson disk sampling within territory bounds
- Minimum distance between points: 8px
- Fill: owner's player color at 40% opacity, circle `r: 1px`

**Pattern Combination:**
- All pattern layers for a territory are rendered as sibling `<g>` elements within the territory group
- Order: Military (bottom), Economic, Cultural, Covert (top)
- Unified territory (one player owns all four dimensions): solid fill of player color, no patterns, additional `stroke: #d4a843` (gold), `stroke-width: 1px`
- Small territory rule: if `d3.geoArea(territory)` < 2000 sq px at default zoom, reduce pattern density by 50% (double all spacing values). If < 1000 sq px, replace patterns with a single 4px colored dot per dimension owner, positioned in the territory's label area.

### 1.5 Territory Name Labels

- Position: territory centroid from `d3.geoCentroid()`
- Text: territory name from GeoJSON properties
- Font: Inter, 9px
- Fill: `#7d827e` (text-secondary) at 40% opacity

### 1.6 Territory Hover Callout

**Content:**

- Four rows, each:
  - Dimension icon: inline SVG, 10px × 10px, owner's player color

**Dismiss:** mouseleave on territory — immediate removal, no delay.

### 1.7 Attack Arrows

## 11. SOUND ENGINE

### 11.1 Initialization

- Single `AudioContext` created on first user interaction (click or keypress)
- Lazy initialization: context created on first sound trigger call
- Module: `soundEngine.ts` exports trigger functions

### 11.2 Sound Definitions

**Card Play:**
- `playCardSound()`: OscillatorNode, type 'sine', frequency 800Hz
- GainNode: start at -18dB, exponential ramp to -60dB over 50ms
- Duration: 50ms total

**Territory Flip:**
- `playTerritoryFlipSound()`: OscillatorNode, type 'sine', frequency 120Hz
- Frequency ramp: 120Hz to 80Hz over 150ms
- GainNode: constant -15dB
- Duration: 150ms total

**Cultural Pressure:**
- `playCulturalPressureSound(level)`: OscillatorNode, type 'sine'
- Level 1 (30% threshold): frequency 200Hz, ramp to 300Hz over 200ms
- Level 2 (40% threshold): frequency 300Hz, ramp to 400Hz over 200ms
- GainNode: constant -28dB (level 1) or -26dB (level 2)
- Duration: 200ms total

**Victory:**
- `playVictorySound()`: three sequential OscillatorNodes
- Tone 1: C5 (523.25Hz), 150ms, -12dB
- Gap: 100ms
- Tone 2: E5 (659.25Hz), 150ms, -12dB
- Gap: 100ms
- Tone 3: G5 (783.99Hz), 150ms, -12dB
- Total duration: 650ms

**Defeat:**
- `playDefeatSound()`: two sequential OscillatorNodes
- Tone 1: G4 (392Hz), 300ms, -15dB
- Gap: 200ms
- Tone 2: C4 (261.63Hz), 300ms, -15dB
- Total duration: 800ms

### 11.3 Trigger Conditions

- Card play: after successful `military_attack`, `economic_invest`, or `deploy_agent` reducer response
- Territory flip: after any `dimension_owner_change` call
- Cultural pressure level 1: when any territory's `influence_pct` crosses above 30%
- Cultural pressure level 2: when any territory's `influence_pct` crosses above 40%
- Victory: on game end with player as winner
- Defeat: on game end with AI as winner
